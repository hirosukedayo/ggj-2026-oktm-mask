# スコア・リザルト判定ロジックの変更

## 概要
現在の「中心からの距離によるスコア（1-3点）」というロジックを廃止し、**「特定の重要箇所（スポット）を撮影できたかどうか」**に基づくイベントフラグ形式に変更する。
エンディング到達条件は「特定の2箇所」を撮影することとし、それ以外の場合は撮影したスポットに応じたエピソード（テキスト）を表示してループ（再挑戦）となる。

## 現状 (As-Is)
- **判定**: 画像中心からのユークリッド距離で判定。
- **スコア**: 距離に応じて 3点, 2点, 1点。
- **結果**: 合計スコア（最大6点）が一定以上ならエンディング。
- **データ**: 写真データには `score` プロパティのみ付与。

## 変更後 (To-Be)
- **判定**: 画面上に定義された複数の「スポット（円形領域）」と撮影範囲の衝突判定。
- **データ**:
  - `Spot` 定義: `{ id, x, y, radius, type, ... }`
  - 写真データ: `spotId` (撮影したスポットのID) を保持。
- **フロー**:
  - 撮影完了後、リスト内の `required` タイプ（特定2箇所）が全て含まれていればエンディングへ。
  - それ以外の場合、撮影したスポットに関連するテキストを表示してリセット（ループ）。
- **スポット構成案 (仮)**:
  - Spot A (Required): エンディング条件1
  - Spot B (Required): エンディング条件2
  - Spot C (Optional): エピソードC
  - Spot D (Optional): エピソードD
  - None: 何も映っていない場合
